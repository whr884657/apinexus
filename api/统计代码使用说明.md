# 本地接口 · 调用统计代码怎么用

面向站长 / 接口作者。把统计代码贴到**本地接口 PHP 文件最上面**（`<?php` 之后、业务代码之前）。

---

## 一、推荐写法（任意目录深度，共 3 行）

无论文件在 `api/`、`api/foo/` 还是 `api/a/b/c/`，都能自动找到站点根目录的 `core/bootstrap.php`：

```php
<?php
for ($d = __DIR__; !is_file($d . '/core/bootstrap.php') && $d !== dirname($d); $d = dirname($d)) {}
require_once $d . '/core/bootstrap.php';
ApiStats::hit();
```

说明：

1. 兼容 PHP 7.4～8.2。
2. 后台「调用地址」请写成与脚本一致的路径，如 `/api/yiyan/v1.php`。
3. 也可写 `ApiStats::hit(12);`（括号内为接口 ID）。

---

## 二、备选：已知目录深度时（2 行）

若你清楚脚本相对站点根的层数，可用 `dirname`（**层数数错会找不到文件**）：

| 脚本位置 | 写法 |
|----------|------|
| `api/xxx.php`（在 api 根下） | `require_once dirname(__DIR__) . '/core/bootstrap.php';` |
| `api/foo/xxx.php`（多一层） | `require_once dirname(__DIR__, 2) . '/core/bootstrap.php';` |
| `api/a/b/xxx.php`（再深一层） | `require_once dirname(__DIR__, 3) . '/core/bootstrap.php';` |

完整两行示例（以 `api/foo/xxx.php` 为例）：

```php
<?php
require_once dirname(__DIR__, 2) . '/core/bootstrap.php';
ApiStats::hit();
```

> 规则：`dirname` 第二个参数 = 从当前文件目录「走到站点根」要上跳的层数。

---

## 三、更短：仅 `api` 下一层子目录时（1 行）

程序自带 `api/hit.php`。若你的脚本在 **`api/某个子目录/`**（如 `api/yiyan/v1.php`），可只写：

```php
<?php
require_once dirname(__DIR__) . '/hit.php';
```

更深目录请改层数，例如 `api/a/b/x.php` 用 `dirname(__DIR__, 2) . '/hit.php'`，或改回第一节的三行写法。

---

## 四、代理接口要不要贴？

**不要。** 对外地址是 `/apis/短码` 的代理接口，由系统网关自动统计。

---

## 五、常见问题

| 现象 | 处理 |
|------|------|
| 调用了但次数不涨 | 先做结构更新；调用地址与脚本路径一致；接口已通过审核且未禁用 |
| 提示维护中 / 不可用 | 后台设为「正常」，并确认已审核通过 |
| 找不到 bootstrap | 优先用第一节三行写法；或核对本节目录层数表 |
