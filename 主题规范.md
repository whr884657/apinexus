# misc-api 主题开发规范

本文档说明前台与用户中心主题的目录结构、自动识别机制及开发约定。管理员后台（`/admin/`）始终使用系统默认 UI，不随前台主题切换。

---

## 一、主题目录位置

所有主题位于：

```
core/theme/{主题ID}/
```

系统在启动时**自动扫描** `core/theme/` 下的一级子文件夹：若目录内存在有效的 `theme.json`，即识别为可用主题，并在后台「主题设置」中展示，**无需手动注册**。

---

## 二、标准目录结构

```
core/theme/{id}/
├── theme.json              # 主题元数据（必填）
├── preview.png             # 后台预览图（开发阶段随主题包一并部署）
├── assets/
│   ├── theme.css           # 前台页面样式（必填）
│   ├── theme.js            # 前台页面脚本（可选）
│   ├── user.css            # 用户中心样式覆盖（可选）
│   └── auth.css            # 登录/注册/忘记密码样式覆盖（可选）
├── layout/
│   ├── header.php          # 前台顶栏 + 抽屉
│   └── footer.php          # 前台页脚
├── pages/                  # 前台页面模板
│   ├── home.php
│   ├── apis.php
│   └── …
└── user/                   # 用户中心主题包（与前台主题联动）
    ├── layout.php          # 用户中心 shell（侧边栏 + 顶栏）
    └── auth/               # 认证页视图
        ├── login.php
        ├── register.php
        ├── forgot.php
        └── bind.php
```

### 资源隔离原则

- 每个主题的 CSS、JS **必须放在该主题 `assets/` 目录内**，禁止写入公共 `assets/css/` 或与其他主题共用文件。
- 前台路由 PHP（如 `index.php`、`apis.php`）仅负责业务逻辑，**视图 HTML 放在主题 `pages/` 中**。
- 用户中心路由 PHP（如 `user/login.php`）仅负责校验与 POST 处理，**视图放在主题 `user/auth/` 中**。

## 三、theme.json 字段

| 字段 | 必填 | 说明 |
|------|------|------|
| `id` | 是 | 与文件夹名一致，小写字母/数字/下划线 |
| `name` | 是 | 后台展示名称 |
| `version` | 否 | 主题包版本 |
| `description` | 否 | 简短说明 |
| `preview` | 否 | 预览图相对路径，默认 `preview.png` |

示例：

```json
{
    "id": "slate",
    "name": "青绿平台",
    "version": "2.2.0",
    "description": "浅色青绿 API 平台风",
    "preview": "preview.png"
}
```

---

## 四、作用范围

| 区域 | 是否随 `frontend_theme` 切换 | 说明 |
|------|------------------------------|------|
| 前台页面 `/`、`/apis` … | **是** | `vs_frontend_page()` → `ThemeManager::renderBody()` |
| 用户中心 `/user/*` | **是** | `ThemeManager::renderUserLayoutStart/End()`、`renderAuthPage()` |
| 管理员后台 `/admin/*` | **否** | 固定系统默认 UI |

切换主题后，用户访问登录页、注册页、控制台等，均加载**当前主题包**内的 `user/` 模板与样式。

---

## 五、预览图

- 路径：`core/theme/{id}/preview.png`
- 在主题开发阶段由开发者随主题包**预先部署**，供后台主题设置页展示。
- 后台读取顺序：`theme.json` 的 `preview` 字段 → 根目录 `preview.png` → 其他常见文件名。

---

## 六、新增主题步骤

1. 在 `core/theme/` 下新建文件夹，命名即主题 ID。
2. 编写 `theme.json`。
3. 实现 `assets/theme.css`、`layout/`、`pages/`（前台）。
4. 实现 `user/layout.php` 与 `user/auth/*.php`（用户中心；可 initially 复用 `default` 再逐步差异化）。
5. 放置 `preview.png`。
6. 后台「主题设置」保存切换，无需改数据库或注册代码。

若某文件在当前主题中缺失，`ThemeManager` 会**回退到 `default` 主题**同名文件（用户中心布局与认证页均支持此机制）。

## 七、现有主题

| ID | 名称 | 前台特点 | 用户中心 |
|----|------|----------|----------|
| `default` | 默认主题 | 浅色卡片 + 右侧抽屉 | 系统默认 admin 风格 shell + 角色动画登录 |

## 八、相关代码入口

| 文件 | 作用 |
|------|------|
| `core/ThemeManager.php` | 扫描、切换、渲染 |
| `core/helpers.php` → `vs_frontend_page()` | 前台页面入口 |
| `user/includes/layout.php` | 用户中心布局委托 |
| `admin/system/theme.php` | 后台主题选择 |
