# ApiNexus 3.10.0 发行说明

**发布日期：** 2026-07-14  
**类型：** 中版本（用户投稿流程、审核三态、库字段与邮件通知）

## 下载

- ZIP：https://gitee.com/xunjinlu/apinexus/releases/download/v3.10.0/apinexus3.10.0.zip
- 标签：`v3.10.0`

## 变更摘要

1. **用户中心「API 管理」**  
   开发者可提交、编辑、删除自有接口；提交后进入待审核。

2. **审核三态**  
   `audit`：待审核 / 通过 / 不通过。管理员在「接口列表」直接发布仍默认通过。

3. **审核原因**  
   新增 `rejectreason`；管理员拒绝时可填原因（选填），用户可在 API 管理页查看。

4. **邮件通知**（需已配置 SMTP）  
   - 开发者提交或重新提交 → 通知全部启用中的管理员邮箱  
   - 管理员通过/不通过 → 通知投稿用户邮箱（不通过时附带原因，若有）

5. **接口审核页**  
   空状态样式对齐「接口列表」；筛选含待审核 / 已通过 / 未通过；拒绝时弹窗填写原因。

6. **其它**  
   清理后台可见文案中的库内数字对照（与《界面勿泄露实现细节》一致）。

## 升级说明

1. 更新至 v3.10.0  
2. **系统管理 → 系统升级**，执行数据库结构更新（`3.10.0`）  
3. 确认系统设置中邮箱发信已配置，否则通知会跳过并提示未送达  
4. 若启用 Redis，结构更新后按需清空业务缓存

## 库变更要点

| 项 | 说明 |
|----|------|
| `api.audit` | 取值含义调整为：0 待审核 · 1 通过 · 2 不通过（升级时原「不通过=0」会迁移为 2） |
| `api.rejectreason` | 新增，审核不通过原因，最长 500 字 |
